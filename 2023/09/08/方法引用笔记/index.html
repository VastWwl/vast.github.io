方法引用

方法引用概述

什么是方法引用？

​ 把已经存在的方法拿过来用，当做函数式接口中抽象方法的方法体

:: 是什么符号？

​ 方法引用符

方法引用时注意什么？

  1. 需要有函数式接口
  2. 被引用的方法必须已存在
  3. 被引用方法的形参与返回值需要跟抽象方法保持一致
  4. 被引用方法的功能必须满足当前的需求

方法引用格式

引用静态方法

格式：类名 :: 静态方法

引用成员方法

格式：

  • 对象 :: 成员方法（其他类）

  • this :: 成员方法（当前类）

  • super :: 成员方法（父类）

注意：

  • 静态方法中无法使用this和super关键字，故不能使用其引用方法

引用构造方法

格式： 类名 :: new

使用类名引用成员方法

格式：类名 :: 成员方法

引用规则：

  1. 引用出需要时函数式接口
  2. 被引用的方法需要已经存在
  3. 被引用的方法的形参，需要跟抽象方法的第二个形参到最后一个形参保持一致，返回值需要保持一致
  4. 被引用方法的功能需要满足当前的需求

抽象方法形参的详解：

  • 第一个参数：
    • 表示被引用的方法的调用者，决定了可以引用那些类中的方法。
    • Stream流中，第一个参数一般都表示流里面的每一个数据。
    • 假设流里面的数据时字符串，那么使用这种方式进行方法引用，只能引用String这个类中的方法
  • 第二个参数到最后一个参数：
    • 跟被引用方法的形参保持一直，如果没有第二个参数，说明被引用的方法需要是无参的成员方法

注意：

  • 不能引用所有类中的成员方法，如果抽象方法中的第一个参数是A类型的那么只能引用A类型中的方法

引用数组的构造方法

格式：数据类型[] :: new

作用：创建一个指定类型的数组

注意：数组的类型，需要跟流中数据的类型保持一致
